# Product Requirement Document (PRD)

## Project: Gulmohar - Hyper-Local Catalog Platform
*   **Target Stack:** Next.js 16.2.9 (App Router), Tailwind CSS v4, Supabase (PostgreSQL, Auth, Storage)
*   **Author:** Software Development Lead / Industry Consultant
*   **Status:** Production-Ready / Approved Scope

---

## 1. Executive Summary & Business Goals
The client operates a physical retail storefront named **Gulmohar**, selling inventory across three distinct physical sections: **Women's Wear, Jewelry, and Cosmetics**. They serve a local offline market and have no existing digital presence, e-commerce infrastructure, or delivery operations.

### Core Objectives:
*   **Digital Window Display:** Establish a high-end, visual digital catalog to capture hyper-local search intent and drive foot traffic into the physical store.
*   **Zero-Friction Inbound Leads:** Eliminate checkout and order management overhead. The website will act as an interface for discovery, funneling high-intent buyers into a direct **WhatsApp Inquiry Pipeline** to verify item availability before visiting.
*   **Streamlined Admin Operations:** Provide a simple internal portal for the store manager to manage the catalog (add, view, and delete items) directly from a smartphone or laptop.

---

## 2. User Personas & Target Audience

### Persona A: The Local Shopper
*   **Behavior:** Local resident browsing clothes, makeup, or jewelry at home. Prefers to inspect items physically before purchasing but wants to confirm selection/stock availability before making the trip to the shop.
*   **Action Pattern:** Discovers a specific product online $\rightarrow$ clicks a direct WhatsApp CTA $\rightarrow$ walks into the store to purchase.

### Persona B: The Store Owner / Manager
*   **Behavior:** Runs day-to-day shop floor operations. Non-technical, requires an administration engine that is fast, robust, and zero-maintenance.
*   **Action Pattern:** Unboxes fresh shop inventory $\rightarrow$ snaps a mobile photo $\rightarrow$ uploads details to the dashboard in under 60 seconds.

---

## 3. Scope of Work & Feature Requirements

### 3.1 Public Storefront (Customer Facing)
*   **Homepage Header / Navigation:** 
    *   Sticky header containing clean brand logo and strict 4-item navigation menu: `Home`, `Women's Wear`, `Jewelry`, `Cosmetics`, `Contact Us`.
*   **Homepage Sections:**
    *   **Hero Visual Banner:** Curated imagery highlighting seasonal trends with immediate `[ Browse Catalog ]` action.
    *   **The Three Pillars:** Clear, high-impact cards dividing the shop into its physical sections: Women's Wear, Jewelry, and Cosmetics. Clicking a card jumps directly to that filtered collection.
    *   **Hyper-Local Map & Hours:** Integrated interactive Google Map embed combined with clear indicators showing current open/closed operational status.
*   **Product Catalog Grid:**
    *   Unified, responsive grid handling heavy image delivery. Items must render clear imagery, titles, pricing (optional/on-request for fine jewelry), and status tags (e.g., `Available`, `Low Stock`).
*   **Product Detail Page (PDP):**
    *   Dedicated high-performance paths displaying expanded description text and clean image galleries.
    *   **The WhatsApp Conversion CTA:** High-visibility layout button reading `[ Enquire Availability via WhatsApp ]`. Clicking triggers a pre-formatted message sent straight to the owner's WhatsApp Business app detailing the item's title and link.

### 3.2 Admin Dashboard (Internal Facing)
*   **Authentication Boundary:** Simple, ironclad login portal restricted via Supabase Auth credentials.
*   **Inventory Ledger Workspace:** Single-pane data layout listing active catalog items, current tags, and simple instant deletion triggers.
*   **Product Ingestion Tool (CRUD):** 
    *   Streamlined form to specify: Title, Description, Category Selection (Restricted dropdown selection to: `womens-wear`, `jewelry`, `cosmetics`), Price, and Image Uploads.
    *   Direct integration with Supabase Storage buckets for multi-image drops.

## 4. Technical Architecture & Data Constraints

### 4.1 Database Design (`products` Table Schema)
```sql
create table products (
  id uuid default gen_random_uuid() primary key,
  title text not null,
  description text,
  category text not null, -- Checked constraints: 'womens-wear', 'jewelry', 'cosmetics'
  price numeric, -- Nullable to support "Price on Request" logic
  image_urls text[] default '{}', -- URLs generated by Supabase Storage
  is_featured boolean default false,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);
